1.4 Miniguia de Estudo (Entrega Final)

Agora, organize o que você aprendeu ao longo do estudo.

Resumo do Tema:
"Git é um sistema de controle de versão distribuído utilizado para gerenciar e versionar o código-fonte de projetos de software. GitHub é uma plataforma baseada na web para hospedar projetos Git e colaborar em equipe."

Glossário:

Commit: Registra mudanças no repositório local.

Clone: Cria uma cópia local de um repositório remoto.

Branch: Linha de desenvolvimento independente.

Merge: Combina alterações de diferentes branches.

Fork: Cria uma cópia de um repositório para contribuir independentemente.

Conjunto de Prompts:

"Como faço um commit no Git?"

"Qual a diferença entre git pull e git fetch?"

"Como crio uma branch no Git?"

"Como resolver conflitos de merge no Git?"
